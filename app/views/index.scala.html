One Messaging Platform, Multiple Recipients

No more clutter from unnecessary messaging applications, pick your favorite and send messages between multiple platforms. We all have a favorite app, why would we need another?

Powered by the Microsoft Bot Framework

By utilizing the powers of Microsoft's bot framework, we were able to seamlessly create multiplatform messaging functionalities. Thanks to the framework Pydgn users can easily communicate with one another regardless of their messaging application of choice.

No App? No Problem.

Thanks to the SMS functionalities that come with the Twilio API, Pydgn users don't even need a messaging platform. By registering their device with Pydgn via SMS, users can still send messages to recipients across all Pydgn supported messaging platforms.

The Beauty Behind the Backend

The Google Cloud platform was the powerhouse behind running the Pydgn backend. The Pydgn backend needs to handle incoming data from multiple messaging transactions, by deploying via Google's Cloud Platform scalability will not be an issue. Regardless of the length or quantity of messages being handled by the backend the GCP is the best bet.

Why Mongo?

MongoDB is a great choice when it comes to picking a database. JSON is a structure we've come to know and love, and Mongo's NoSQL database makes us feel right at home. Since Pydgn would ideally have a user base of all supported messaging platforms combined, scalability is very important. MongoDB provides Pydgn with all the functionality it needs in a database, and none of the functionality it doesn't.
